Привет, писал тебе недавно насчет ТЗ. Наконец обсудили этот вопрос. Идея такова: написать веб приложение для покупки, создания и поиска авиабилетов. В итоге должно получится веб приложение с REST API. Пункты:
1) Создать сущности билет, клиент, рейс. Реализовать возможность создавать билеты,
 возможность получать весь список билетов и возможность получать конкретный билет по id.
2) Возможность бронировать билеты для конкретного пользователя. Возможность фильтровать список билетов
по определенным параметрам. Начисление бонусов клиенту при покупке билета и возможность применения
этих бонусов при следующих покупках.
3) На вход подается количество имеющихся самолётов, количество мест в каждом и популярность
определенных маршрутов. Нужно на основе имеющихся данных наиболее оптимально сгенерировать
определенное количество билетов на каждый рейс и периодичность рейсов. Входные данные можно
брать из конфиг файла/из базы или из другого источника на твой выбор, главное чтобы было консистентно.
